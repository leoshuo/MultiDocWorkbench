# Document Workspace 智能文档工作台

> 让文档处理从重复劳动变成一键复现

---

## 一句话介绍

**Document Workspace** 是一个能够"记住"你如何处理文档、并在下次遇到类似任务时自动帮你完成的智能助手。

---

## 产品解决什么问题？

### 场景一：重复的日报汇总

小王是部门主任秘书，每天需要把 10+ 份员工日报合并成一份领导版日报。流程总是相似的：

1. 打开每份日报，提取关键信息
2. 按照固定格式整理大纲
3. 合并重复内容，精简表述
4. 输出最终文档

**以前**：每天重复这套流程，耗时 2 小时  
**现在**：第一次操作时系统自动"记住"，之后只需点一个按钮，5 分钟出结果

### 场景二：复杂的文档分析

小李需要分析竞品报告，每次都要：

1. 先抽取文档大纲结构
2. 针对不同章节执行不同处理
3. 最后汇总生成分析报告

**以前**：每份报告都要从头操作一遍  
**现在**：把处理流程"沉淀"下来，新报告来了一键分析

## 核心能力

### 1. 智能大纲抽取

上传任意文档（支持 txt、md、docx），AI 自动分析文档结构，生成 1-5 级层级大纲。

```
原始文档 → AI 分析 → 结构化大纲
           （几秒钟）
```

**特点**：
- 自动识别标题层级
- 每个章节生成摘要
- 支持手动调整和编辑
- 历史大纲自动存档，随时恢复

### 2. 经验沉淀 —— 系统的"记忆力"

这是本产品最核心的能力。

**什么是"沉淀"？**

就像你教一个新同事处理文档：
- 你演示一遍怎么做
- 他记住了步骤
- 下次类似任务他就会做了

系统也是这样：
- 你操作时，系统在"看"和"记"
- 记下来的叫做"沉淀记录"
- 下次遇到类似任务，系统按照记录自动执行

**沉淀记录什么？**

| 记录项 | 说明 | 举例 |
|--------|------|------|
| 输入来源 | 基于什么内容操作 | 从员工日报第二段提取信息 |
| 动作执行 | 执行什么操作 | 提取人员姓名，去除职务称谓 |
| 记录位置 | 结果写入哪里 | 写入"参会人员"标题下的摘要 |
| 执行摘要 | 操作结果概要 | 已提取 5 个姓名 |

### 3. 一键 Replay —— 沉淀的价值体现

沉淀下来的经验，可以：

- **单步 Replay**：重新执行某一步操作
- **整体 Replay**：按顺序执行所有步骤
- **沉淀集 Replay**：多个沉淀组合，完成复杂任务

**Replay 有两种模式**：

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| 大模型模式 | 智能理解意图，允许适应新内容 | 文档格式略有变化 |
| 脚本模式 | 严格按照记录执行，一字不差 | 格式固定的标准文档 |

**智能之处**：大模型模式下，即使新文档和之前的不完全一样，系统也能理解你的意图，找到对应位置完成操作。

### 4. 双工作台设计

| 工作台 | 面向人群 | 功能定位 |
|--------|----------|----------|
| 多文档处理工作台 | 普通用户 | 简洁界面，一键执行 |
| 经验沉淀工作台 | 配置人员 | 录制沉淀、配置流程 |

**设计理念**：
- 配置人员在后台把流程配好
- 普通用户在前台点击按钮就能用
- 所有数据全局共享，一处上传两边生效

## 典型使用流程

### 第一阶段：配置（一次性）

```
配置人员在"经验沉淀工作台"操作：

1. 上传样本文档
2. 点击"自动沉淀"开始录制
3. 按正常流程处理文档（抽取大纲、执行指令、填入摘要...）
4. 点击"结束沉淀"
5. 生成沉淀记录，可编辑调整
6. 创建沉淀集，关联到前台按钮
```

### 第二阶段：使用（日常）

```
普通用户在"多文档处理工作台"操作：

1. 上传待处理文档（或系统自动从配置目录加载）
2. 点击对应的功能按钮
3. 系统自动执行 Replay
4. 查看处理结果
```

## 技术亮点

### AI 驱动的文档理解

- **大纲智能抽取**：AI 分析文档结构，准确识别层级关系
- **指令智能执行**：理解自然语言指令，完成复杂文档操作
- **Replay 意图分析**：智能匹配目标位置，即使文档变化也能正确定位

### 双记录机制

每个操作步骤同时保存两份记录：

| 记录类型 | 用途 |
|----------|------|
| 大模型记录（llmScript） | 大模型 Replay 时使用，支持泛化 |
| 脚本记录（originalScript） | 脚本 Replay 时使用，精确复现 |

两套记录互为备份，确保系统稳定可靠。

### 容错与回退

```
大模型执行 → 成功 → 完成
     ↓
   失败
     ↓
脚本执行 → 成功 → 完成
     ↓
   失败 → 报告原因，跳过此步
```

系统自动在两种模式间切换，最大程度保证任务完成。

| 本地运行 | 开发测试 |

支持连接外部 AI 服务，也支持接入本地私有化模型。
